Moduł 2, Zadanie 2.0: Setup repozytorium i Deployment

Witamy w Module 2! Tym razem będziemy łączyć się z prawdziwymi serwerami (API) w internecie, aby pobierać dane. Aby nasza aplikacja była w pełni profesjonalna, będziemy pracować na sklonowanym repozytorium kurs-react. Nauczymy się też, jak sprawić, by kod z naszego komputera automatycznie stawał się stroną widoczną dla całego świata pod Twoim własnym linkiem na GitHub Pages!

1. Prawidłowa struktura projektu

Otwórz folder swojego repozytorium kurs-react w edytorze Visual Studio Code. Profesjonalne projekty nie trzymają wszystkich plików "luzem" w jednym miejscu. Zbudujmy porządek!

Schemat: Tak powinna wyglądać standardowa struktura plików statycznych (HTML, CSS, JS).

Stwórz w swoim repozytorium następujące foldery i pliki:

📁 kurs-react/ (Główny folder repozytorium) ├── 📄 index.html <-- Główny plik Twojej strony ├── 📁 css/ │ └── 📄 style.css <-- Tutaj będą style └── 📁 js/ └── 📄 app.js <-- Tutaj będziemy pisać nasz kod JS

2. Startowy plik index.html

Wklej poniższy kod do swojego nowo utworzonego pliku index.html. To szkielet, na którym będziemy budować aplikacje w tym module.

<!DOCTYPE html> <html lang="pl"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Moduł 2: Moja Aplikacja API</title> <!-- Podpięcie zewnętrznego pliku CSS z folderu css --> <link rel="stylesheet" href="./css/style.css"> </head> <body> <header> <h1>Moduł 2: Interakcja z serwerem</h1> </header> <main id="kontener-aplikacji"> <h2>Miejsce na projekt...</h2> <p>Niedługo pojawią się tu dane pobrane z prawdziwego serwera!</p> </main> <!-- Podpięcie skryptu JS z folderu js jako MODUŁ --> <script type="module" src="./js/app.js"></script> </body> </html>

3. Zapisz i wyślij na serwer (Push)

Teraz użyjemy magii Gita, aby zapisać te pliki i wysłać je do repozytorium w chmurze.

# 1. Dodaj wszystkie nowe pliki do "poczekalni" git add . # 2. Stwórz "zapis gry" (commit) z opisem git commit -m "Inicjalizacja struktury projektu dla Modulu 2" # 3. Wyślij pliki do GitHuba (używając skonfigurowanego wcześniej SSH) git push origin main

4. Publikacja w internecie (GitHub Pages Deployment)

Nasz kod jest już w chmurze, ale teraz powiemy GitHubowi: "Hej, potraktuj mój plik index.html jako prawdziwą stronę i udostępnij ją światu!"

Schemat: Gdzie w GitHubie kliknąć, aby włączyć Pages.

1 Wejdź w ustawienia Repozytorium

Na stronie swojego repozytorium na GitHubie (tam, gdzie widzisz swoje pliki), kliknij zakładkę Settings (ikonka zębatki) na górnym pasku.

2 Zakładka Pages

Zjedź nieco niżej po lewym menu i kliknij opcję Pages.

3 Wybierz źródło (Source)

W sekcji "Build and deployment", pod napisem Branch (Gałąź), zobaczysz przycisk z napisem None. Kliknij go i zmień na main.

Obok pojawi się folder (zazwyczaj / (root)). Zostaw to bez zmian i kliknij Save (Zapisz).

I to wszystko! Co się teraz stanie?
Po kliknięciu "Save", GitHub potrzebuje około 1-2 minut na zbudowanie Twojej strony. Kiedy skończy, na samej górze zakładki Pages pojawi się powiadomienie z linkiem w stylu:
https://twoja-nazwa.github.io/kurs-react/.

Najlepsze jest to, że ten proces ustawiłeś tylko raz! Od teraz za każdym razem, gdy użyjesz komendy git push na swoim komputerze, GitHub automatycznie i bez Twojego udziału zaktualizuje tę stronę w internecie.
